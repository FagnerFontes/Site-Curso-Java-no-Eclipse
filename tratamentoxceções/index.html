Tratamento de Exceções


Exceções

Ao executar o código Java, diferentes erros podem acontecer:

  • erros de codificação feitos pelo programador
  • erros devido a entrada ou outros erros imprevistos.
Quando ocorre um erro, o Java normalmente para e gera uma mensagem de erro. O termo técnico para isso é: Java lançará uma exceção jogará um erro.

De forma interpretativa em Java, um erro é algo irreparável, a aplicação trava ou encerrada drásticamente. Já exceções é um fluxo inesperado da nossa aplicação exemplo:

  • Querer dividir um valor por zero.
  • Querer abrir um arquivo por zero.
  • Abrir uma conexão de banco, com usuário e/ou senha inválida
Todos estes cenários e os demais, não são erro mas sim fluxos, são previstos pela aplicação. É ai que entra uma responsabilidade do desenvolvedor, prever situações iguais a estas e realizar o que denominamos de Tratamento de Exceções.

  1. Hierarquia das Exceções:
    • Todas as exceções em Java são derivadas da classe Throwable. Existem duas subclasses de Throwable
      • .
      1. Exception : Representa condições que um aplicativo pode querer capturar e tratar. Dentro Exception, há duas categorias:
        • Dentro de Excption, há duas categorias:
          • checked exceptions: São verificadas em tempo de compilação. O compilador força o programa a tratar exceções. Ex.: IOException, SQLException
          • unchecked exceptions (exceções não verificadas): (exceções não verificadas): Não são verificadas em tempo de compilação, são deverivadas de RuntimeException. Ex:. NullPointerException, ArrayIndexOutofBoundsException.
  2. Blocos Try-Catch-Finally
    • try : O bloco de código que pode gerar uma exceção é colocado dentro do bloco try
    • catch: Captura e trata a exceção. Pode haver múltiplos blocos catch para capturar diferentes tipos de exceções.
    • finally: Um bloco opcional que sempre é executado, quer a exceção ocorra ou não. É útil para liberar recursos, como fechar arquivos ou conexão de banco de dados.
  3. Throws :
    A palavra-chave throws é usada para indicar que ele pode gerar exceções verificadas, que devem ser tratadas ou declaradas pela classe que chama o método.
