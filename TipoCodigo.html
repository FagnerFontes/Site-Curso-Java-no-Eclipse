A linguagem interpretada X e a linguagem compilada.

Os termos "código-compilado" e "Código-interpretado" referem-se a duas abordagens diferentes para a execução de programas de compútador. Vamos explorar cada uma delas em detalhes.

  1. Cóndigo Compilado: O código compilado é transformado de alto nível para código de máquina por um compilador. Esse código de máquina pode ser executado diretamente pelo processador do computador.

Características

  1. Desempenho:

    Normalmente, programas compiladps executam mais rapidamente porque a tradução para o código de máquina já foi feita antes da execução.

  2. Processo de Compilação:

    Requer uma etapa de compilação separada antes da execução do programa. Isso pode adicionar tempo de desemvolvimento.

  3. Erro de Tempo de Compilação:

    Muitos erros podem ser detectados durante a fase de compilação, antes mesmo do programa ser executado.

  4. Distribuição:

    Distribuir programas compilados geralmente envolve a distribuição de código binário.

    • C
    • C++

Código Interpretado

Definição

    O código interpretado é executado linha por linha por um interpretador. Ao invés de ser transformado em código de máquina antes da execução, o interpretado lê e executa o código-fonte diretamente.

Características:

  1. Desempenho:

    Geralmente, programas interpretados são mais lentos do que programas compilados devido à sobrecarga do processo de interpretação.

  2. Flexibilidade:

    Não requer uma etapa de compilação separada; o código pode ser executado imediatamente.

  3. Erro de Tempo de Execução:

    Muitos erros só são detectados durante a execução, o que pode tornar a depuração mais difícil.

  4. Interatividade:

    Ideal para linguagens de script e ambientes interativos, permitindo mudanças rápida e testes imediatos.
